# Interactive Examples

Try out common 1sh API use cases with real examples you can test directly in your browser.

## Quick Start Examples

### Example 1: Create a Simple Short Link

The most basic use case - shorten a long URL.

<ApiEndpoint
  method="POST"
  path="/api/v1/link"
  description="Create a short link for a long URL. This example uses a sample Turnstile token - in production, you'll need to implement Turnstile verification."
  requestBody={{
    type: "object",
    properties: {
      url: {
        type: "string",
        required: true,
        description: "The URL to shorten",
        example: "https://docs.google.com/document/d/1BxiMVs0XRA5nFMdKvBdBZjgmUUqptlbs74OgvE2upms/edit"
      },
      turnstileToken: {
        type: "string",
        required: true,
        description: "Turnstile verification token (use 'demo-token' for testing)",
        example: "demo-token"
      }
    }
  }}
  responseSchema={{
    type: "object",
    properties: {
      success: { type: "boolean" },
      data: {
        type: "object",
        properties: {
          id: { type: "string" },
          shortCode: { type: "string" },
          shortUrl: { type: "string" },
          originalUrl: { type: "string" },
          managementToken: { type: "string" },
          createdAt: { type: "string" }
        }
      }
    }
  }}
/>

### Example 5: Retrieve Multiple Links

## Analytics Examples

### Example 6: Get Basic Analytics

Retrieve click analytics for a specific link.

<ApiEndpoint
  method="GET"
  path="/api/v1/analytics/[linkId]"
  description="Get comprehensive analytics for a link. Use the link ID and management token from a previously created link."
  parameters={[
    {
      name: "linkId",
      type: "string",
      required: true,
      description: "Link ID from link creation response",
      example: "abc123def456"
    },
    {
      name: "token",
      type: "string",
      required: true,
      description: "Management token from link creation",
      example: "mgmt_abc123def456"
    },
    {
      name: "period",
      type: "string",
      required: false,
      description: "Analytics time period",
      example: "7d"
    }
  ]}
  responseSchema={{
    type: "object",
    properties: {
      success: { type: "boolean" },
      data: {
        type: "object",
        properties: {
          link: {
            type: "object",
            properties: {
              id: { type: "string" },
              shortCode: { type: "string" },
              originalUrl: { type: "string" }
            }
          },
          analytics: {
            type: "object",
            properties: {
              totalClicks: { type: "number" },
              uniqueClicks: { type: "number" },
              clicksByDay: { type: "array" },
              topCountries: { type: "array" },
              topDevices: { type: "array" }
            }
          }
        }
      }
    }
  }}
/>

## Integration Workflows

### Workflow 1: Complete Link Creation and Management

1. **Check alias availability** (if using custom alias)
2. **Create the link** with your desired settings
3. **Store the management token** for future operations
4. **Monitor analytics** to track performance

### Workflow 2: Bulk Link Management

1. **Create multiple links** and collect management tokens
2. **Retrieve all links** using the bulk endpoint
3. **Get analytics** for high-performing links
4. **Update or recreate** expired links as needed

### Workflow 3: Campaign Tracking

1. **Create branded links** for each campaign
2. **Set expiration dates** for time-limited campaigns
3. **Monitor real-time analytics** during campaign
4. **Export data** for reporting and analysis

## Testing Tips

- Use `demo-token` as the Turnstile token for testing (production requires real Turnstile implementation)
- Save management tokens from link creation to test analytics endpoints
- Try different alias formats to understand validation rules
- Test error scenarios by using invalid data
- Use the browser's network tab to see actual HTTP requests and responses
